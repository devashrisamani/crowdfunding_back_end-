## Planning

### Concept/Name

**Leaf a Mark** is a crowdfunding platform dedicated to environmental and sustainability initiatives. The platform connects passionate individuals and organizations with community-driven projects that make a positive impact on our planet. From tree planting drives and ocean cleanups to wildlife conservation, Leaf a Mark empowers people to support causes they care about.

### Intended Audience/User Stories

**Primary Users:**

- **Environmental Organizations**: Non-profits, community groups, and activists who want to raise funds for sustainability projects
- **Eco-conscious Individuals**: People who want to support environmental causes and make a difference in their communities
- **Community Groups**: Local organizations organizing cleanups, tree planting, recycling programs, etc.

**User Stories:**

- As an environmental organization, I want to create a fundraiser for my beach cleanup project so that I can raise funds for equipment and supplies
- As a supporter, I want to browse sustainability fundraisers by category so I can find projects that align with my interests
- As a fundraiser owner, I want to update my fundraiser details so I can keep supporters informed about progress
- As a supporter, I want to make a pledge to a fundraiser so I can contribute to environmental causes I care about
- As a fundraiser owner, I want to see all pledges made to my fundraiser so I can track progress toward my goal

### Front End Pages/Functionality

- **Home/Browse Page**

  - Display all active sustainability fundraisers
  - Filter by category (Ocean, Reforestation, Waste & Recycling, etc.) - in progress
  - Search functionality to find specific projects - in progress
  - View fundraiser cards with title, image, goal, and progress
  - Sort by newest, most funded, or closest to goal - in progress

- **Fundraiser Detail Page**

  - Full fundraiser information (title, description, goal, image)
  - Progress bar showing amount raised vs. goal - in progress
  - List of all pledges (with option to show/hide anonymous pledges)
  - Make a pledge form (amount, comment, anonymous option)
  - Owner can choose to hide a specific comment and supporter can choose to delete their comment but no changes can be made to the pledge by the supporter other than that
  - Owner can edit/delete their fundraiser
  - Display category and creation date

- **Create Fundraiser Page**

  - Form to create new fundraiser (title, description, category, location, goal, image URL, date created, open/closed, days remaining)
  - Only accessible to authenticated users
  - Owner automatically set to logged-in user

- **User Profile Page**

  - View user's created fundraisers
  - View user's pledges
  - User registration and login functionality

- **Login/Register Page**
  - User registration form
  - Login form with token authentication
  - Token stored for authenticated requests

### ERD Diagram

The database is structured around three core entities: Users, Fundraisers, and Pledges.

![ERD Diagram](ERD.png)

**Relationships:**

- One User can create many Fundraisers (one-to-many via `owner` ForeignKey)
- One Fundraiser can have many Pledges (one-to-many via `fundraiser` ForeignKey)
- One User can make many Pledges (one-to-many via `supporter` ForeignKey)
- When a Fundraiser is deleted, all its Pledges are automatically deleted (CASCADE)
- When a User is deleted, all their Fundraisers and Pledges are deleted (CASCADE)
